## 第4页 | Agent 核心流程（重点页）

**RAG Agent = 检索(Retrieve) + 增强(Augment) + 生成(Generate)**

```
用户提问: "舰艇消防系统设计有什么强制要求？"
    │
    ▼
① 查询向量化  ──── BGE-base-zh-v1.5 768维嵌入
    │
    ▼
② 混合检索  ──── 语义向量(HNSW) + 关键词(ILIKE)
    │             └─ RRF融合排序 + 文档状态加权(现行>修订>废止)
    ▼
③ Top-K 召回  ──── 检索最相关的5-10个文本片段
    │
    ▼
④ 上下文拼接  ──── Prompt = 系统指令 + K个文档片段 + 用户问题
    │
    ▼
⑤ LLM 生成  ──── 通义千问 qwen-plus 流式输出
    │             └─ System Prompt: "严禁编造，强制标注标准编号+条款"
    ▼
⑥ 引用溯源  ──── 回答附带 [HJB 590B-2025] 第3章第2节 第12条
```

## 第5页 | 混合检索详解

**双通道融合，互补长短**

| 通道 | 技术 | 优势 | 劣势 |
|------|------|------|------|
| 语义检索 | pgvector HNSW + Cosine | 理解"灭火"≈"消防" | 专业编号匹配弱 |
| 关键词检索 | PostgreSQL ILIKE | 精确匹配"HJB 590B" | 不理解语义 |

**RRF 融合公式**: score = 0.7/(60+rank_sem) + 0.3/(60+rank_kw)

**文档状态加权**: 现行有效×1.0 > 修订中×0.8 > 废止×0.5

## 第6页 | 提示词工程

**核心约束（System Prompt 设计）**:

1. **严禁编造**: "严格依托提供的文档内容作答，没有依据则回复'暂无相关标准依据'"
2. **强制溯源**: "回答必须标注引用来源：标准编号 + 具体条款条目"
3. **区分条款性质**: 强制执行 / 推荐执行 / 指导性说明
4. **海军公文风格**: 用语规范、逻辑严密、不使用口语化表达

**多轮对话**: 保留最近3轮历史，延续问答上下文

## 第7页 | 文档处理流水线

```
上传 PDF/DOCX/TXT
    │
    ├── 文字型 ──→ pdfplumber 提取文字 (跳过表格加速60%)
    │
    ├── 扫描版 ──→ 自动检测(平均<30字/页) → Tesseract OCR
    │               └─ 150DPI + 中文语言包 + 4线程并行
    ▼
清洗降噪 ──── 去页眉页脚、页码、水印、空行
    │
    ▼
提取元数据 ──── 标准编号(HJB 590B-2025)、名称、领域、日期、归口单位
    │
    ▼
章节分块 ──── 按"第X章/第X节/第X条"切分，500-800字/块，100字重叠
    │
    ▼
向量化入库 ──── BGE嵌入(768维) + execute_values 批量写入 pgvector
```
